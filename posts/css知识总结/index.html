浏览器渲染过程

三棵树,三步骤

三棵树:

  1. HTML树(DOM)

  2. CSS树(CSSOM)

  3. 上面两颗树合在一起就是render tree

三步骤:

  1. 布局

  2. 绘制

  3. 合成

用JS更新样式时会发生什么?

  1. 执行JS
  2. 算出修改的样式是什么
  3. 布局
  4. 绘制
  5. 合成

三步骤不一定全部执行

不需要绘制的情况 单纯的改变颜色,布局没有变化,这个比较好理解

奇怪是是只需要合成的情况,不一样的浏览器对不同的样式改变有不同的处理方式 具体可以看这个网站csstriggers.com
对于绘制也是这个样子的

CSS动画

transition和animation

关于transition和animation,
我觉得transition比较适合做那种点到点的动画,也就是从一种形态变成另一种形态
而animation适合做中间态变化多的动画,因为它可以定义keyframe更精确的控制时间线而不需要JS的帮助
